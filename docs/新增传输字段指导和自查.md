# 新增传输字段指导和自查

## 字段传递流程

### 方式一：Native → Flutter（通过 Android 配置）

```
xchatkit_config.json
       ↓
XChatKitConfig.java (读取配置)
       ↓
FlutterDemoActivity.java (构建mediaInfoJson)
       ↓
SDLActivityAdapter.java (传递给Flutter)
       ↓
Flutter: XChatEventChannel / XChatMethodChannel
       ↓
RoomBlocV2 (接收并处理)
       ↓
RoomStateV2 (存储状态)
       ↓
HttpClientHelper (使用字段)
```

### 方式二：Flutter 本地配置（独立运行模式）

```
app_config.json / app_config_debug.json
       ↓
AppConfig.fromJson() (读取Flutter配置)
       ↓
room_modules.dart (提取配置值)
       ↓
registerRoomDependencies() (传递到ServiceLocator)
       ↓
service_locator.dart (注册到RoomBlocV2)
       ↓
RoomBlocV2 (接收并处理)
       ↓
RoomStateV2 (存储状态)
       ↓
RoomClientEntranceV2 (使用字段)
```

## 新增字段修改清单

### 1. Android 配置文件（源头）

**文件**: `MyApplicationForFlutter/rsc-sdk/src/main/assets/xchatkit_config.json`

在 `mediaInfo` 或 `userData` 对象中添加新字段。

```json
"mediaInfo": {
  "newField": "默认值"
}
```

---

### 2. Flutter 本地配置文件（独立运行模式）

**文件**: `flutter_module/assets/config/app_config.json` 或 `app_config_debug.json`

在 `mediaInfo` 对象中添加新字段：

```json
"mediaInfo": {
  "newField": "默认值"
}
```

---

### 3. Flutter 配置读取类

**文件**: `flutter_module/lib/config/app_config.dart`

需要修改三个地方：

**a) 添加静态常量默认值**

```dart
static const String _defaultNewField = '/gateway/new-field/path';
```

**b) 添加成员变量**

```dart
final String newField;
```

**c) 在构造函数参数中添加**

```dart
required this.newField,
```

**d) 在 factory defaults() 中添加**

```dart
newField: _defaultNewField,
```

**e) 在 factory fromJson() 中解析**

```dart
final newField = (mediaInfo['newField'] as String?)?.trim();
...
newField: newField?.isNotEmpty == true ? newField! : _defaultNewField,
```

---

### 4. Android 配置读取类

**文件**: `MyApplicationForFlutter/rsc-sdk/src/main/java/com/yc/rtc/rsc_sdk/XChatKitConfig.java`

需要修改两个地方：

**a) 添加成员变量和getter/setter（如果是 mediaInfo 字段）**

```java
// 在 MediaInfo 内部类中添加
private String newField;
public String getNewField() { return newField; }
public void setNewField(String newField) { this.newField = newField; }
```

**b) 解析JSON赋值**

```java
// 在 MediaInfo.fromJSONObject() 方法中添加
mediaInfo.setNewField(json.optString("newField", ""));
```

---

### 5. Android 数据构建类（传递给Flutter的关键步骤）

**文件**: `MyApplicationForFlutter/rsc-sdk/src/main/java/com/yc/rtc/rsc_sdk/FlutterDemoActivity.java`

在 `initXchatData()` 方法中，将字段加入 `mediaInfoJson` 或 `userDataJson`：

```java
// mediaInfo 字段
mediaInfoJson.put("newField", mediaInfo.getNewField());

// 或 userData 字段（使用 userDataJson）
userDataJson.put("newField", "值");
```

⚠️ **常见遗漏**：添加 `XChatKitConfig.java` 的读取后，忘记在此处添加传递代码。

---

### 6. Flutter 配置文件提取（room_modules.dart）

**文件**: `flutter_module/lib/screens/room/room_modules.dart`

在 `_prepareAndRegisterDependencies()` 方法中提取配置：

```dart
// 提取新字段
final newField = _extractValue(args, mediaInfo, 'newField') ?? config.newField;
```

然后在调用 `registerRoomDependencies()` 时传递：

```dart
registerRoomDependencies(
  ...
  newField: newField,
  ...
);
```

---

### 7. Flutter ServiceLocator 注册

**文件**: `flutter_module/lib/di/service_locator.dart`

在 `registerRoomDependencies()` 函数中添加参数：

```dart
void registerRoomDependencies({
  ...
  String? newField,
  ...
}) {
  ...
  sl.registerSingleton<RoomBlocV2>(RoomBlocV2(
    ...
    newField: newField,
    ...
  ));
}
```

---

### 8. Flutter MethodChannel 定义（如需新增MethodChannel方法）

**文件**: `flutter_module/lib/features/xchatkit_adapter/channels/xchat_method_channel.dart`

如果需要通过 MethodChannel 调用 Android 方法：

```dart
static Future<void> newMethod(String param) async {
  await _channel.invokeMethod('newMethod', param);
}
```

对应的 Android 端 Handler 在 `SDLActivityAdapter.java`。

---

### 9. Flutter 状态类

**文件**: `flutter_module/lib/features/room/bloc/room_state_v2.dart`

在状态类中添加字段：

```dart
final String? newField;

RoomStateV2 copyWith({
  ...
  String? newField,
}) => RoomStateV2(
  ...
  newField: newField,
);
```

还需要在 `factory RoomStateV2.initial()` 中添加参数传递：

```dart
factory RoomStateV2.initial({
  ...
  String? newField,
  ...
}) {
  return RoomStateV2(
    ...
    newField: newField,
    ...
  );
}
```

---

### 10. Flutter BLoC 处理

**文件**: `flutter_module/lib/features/room/bloc/room_bloc_v2.dart`

在构造函数中接收参数并传递给 state：

```dart
RoomBlocV2({
  ...
  String? newField,
}) : super(RoomStateV2.initial(
  ...
  newField: newField,
  ...
));
```

或者在 factory 方法中提取：

```dart
final newField = _getValue(args, mediaInfo, 'newField');
```

---

### 11. Flutter 使用类

**文件**: `flutter_module/lib/plugin/common/http_client_helper.dart`

如果字段在 `HttpClientHelper` 中使用：

```dart
final String? newField;

// 在 apiPath 或其他方法中使用
String get apiPath => newField ?? '';
```

---

### 12. Flutter 调用方

**文件**: `flutter_module/lib/features/signaling/room_client_entrance_v2.dart`

添加 getter 访问：

```dart
String? get newField => roomState.newField;
```

在需要的地方使用：

```dart
if (newField != null && newField!.isNotEmpty) {
  // 使用 newField
}
```

## 本次 `picUrl` 遗漏原因

本次问题出现在第3步，`FlutterDemoActivity.java` 中 `mediaInfoJson` 构建时遗漏了 `picUrl`：

```java
// ✅ 已添加
mediaInfoJson.put("mediaEntranceUrl", mediaInfo.getMediaEntranceUrl());
mediaInfoJson.put("mgwUrl", mediaInfo.getMgwUrl());
mediaInfoJson.put("roomMode", mediaInfo.getRoomMode());
// ... 其他字段 ...

// ❌ 遗漏了
mediaInfoJson.put("picUrl", mediaInfo.getPicUrl());
```

---

## 本次 `logUrl` 遗漏原因（Flutter 独立配置方式）

本次问题出现在 Flutter 配置传递链，`room_modules.dart` 中遗漏了 `logUrl` 的提取和传递：

```dart
// ❌ 遗漏了
final logUrl = _extractValue(args, mediaInfo, 'logUrl') ?? config.logUrl;

// ✅ 已添加
registerRoomDependencies(
  ...
  logUrl: logUrl,
  ...
);
```

同时 `service_locator.dart` 也遗漏了参数接收和传递。Fluttter 配置传递链比 Native 方式多了 `room_modules.dart` 和 `service_locator.dart` 两个环节，容易遗漏。
